# MOONDROP Pudding 实测发现

## 设备信息
- 型号: MOONDROP Pudding (MD-TWS-056)
- 芯片: Zhuhai Jieli (杰理) VID=0x05D6, PID=0x000A
- 蓝牙名: MOONDROP Pudding / TWS-056-BLE (BLE)
- 固件版本: 3.5.2
- MAC 地址: D2:D1:13:AB:67:2

## 协议格式

### SPP (GAIA 格式)
```
TX/RX: FF 04 [Len:2 BE] [Seq:1] [Vendor:1] [FeatureID:1] [CmdID:1] [Payload...]
Len = 1 + 1 + payload.size (feature + cmd + payload)
总包长 = 8 + payload.size
```

### 响应格式
- QUERY 响应: payload=[当前值] (1字节)
- SET 响应: payload=[当前值, ?, ?] (3字节，[1][2]未知)

### BLE (RCSP 协议)
- L2CAP CID: 0xC3FF-0xFFFF (动态)
- 数据以 `00 19` 开头
- 不是标准 GATT，是杰理私有协议
- GATT UUID (OTA): `0000ae00/ae01/ae02`

## SPP 已确认功能 (btsnoop 抓包验证)

协议结构: Feature=0x00 用于基础查询, Cmd 为功能号。ANC/Gain/Codec 使用独立 Feature ID。

| 功能 | Feature | Cmd | 结果 |
|------|---------|-----|------|
| 支持的命令 | 0x00 | 0x01 | ✓ 命令列表 |
| 固件版本 | 0x00 | 0x05 | ✓ "3.5.2" |
| EQ查询 | 0x00 | 0x07 | ✓ payload=[子命令] |
| 配置查询 | 0x00 | 0x0C | ✓ payload=[配置ID] |
| 设备状态 | 0x00 | 0x0D | ✓ payload=[07,00,00,00,04] |
| 序列号 | 0x00 | 0x14 | ✓ ASCII |
| 设备ID | 0x00 | 0x15 | ✓ ASCII |
| **Gain查询** | **0x1E** | **0x01** | **✓ 增益级别** |
| **Gain设置** | **0x1E** | **0x02** | **✓ 确认** |
| **ANC 查询** | **0x40** | **0x03** | **✓ payload=[模式]** |
| **ANC 设置** | **0x40** | **0x04** | **✓ 确认** |
| **ANC 可用模式** | **0x40** | **0x29** | **✓ [1,1,1,1,1]** |
| 编解码器查询 | 0x20 | 0x05 | ✓ LDAC状态 |
| 电量 | 0x00 | 0x01 | ✗ 无响应 |

## ANC 模式 (实测确认)

| 值 | 模式 | 状态 |
|----|------|------|
| 0x00 | 关闭 | ✓ 可用 |
| 0x02 | 通透 | ✓ 可用 |
| 0x04 | 降噪 | ✓ 可用 |
| 0x08 | 自适应 | ✗ 暂不可用 |
| 0x10 | 抗风噪 | ✗ 暂不可用 |

## Gain 级别 (实测确认)

| 值 | 级别 |
|----|------|
| 0x00 | 高 |
| 0x01 | 中 |
| 0x02 | 低 |

## BLE 分析 (btsnoop 日志)
- 988 个 BLE 包 (218 TX, 770 RX)
- 所有包以 `00 16` 开头 (RCSP 协议头)
- L2CAP CID: 0xC3FF-0xFFFF
- 无标准 GAIA 格式 (FF 04)
- 无标准 GATT 格式

## 杰理 BLE GATT (OTA 用)
- Service: `0000ae00-0000-1000-8000-00805f9b34fb`
- Write: `0000ae01` (write without response)
- Notify: `0000ae02` (notify)

## 已知问题
1. 部分蓝牙适配器 (MediaTek) RFCOMM 驱动异常，无法创建虚拟串口
2. BLE 连接从 PC 超时，无法探测 GATT 服务
3. ANC 自适应/抗风噪子模式暂不可用
4. 电量查询无响应
5. SET 响应的 [1][2] 字节含义未知

## btsnoop 抓包验证 (2026-07)

通过 Android 手机抓取 MOONDROP Link APP 的 btsnoop HCI 日志，验证了完整的 SPP 通信流程。

### 连接初始化序列

1. 查询支持的命令列表 (Feature=0x00, Cmd=0x01)
2. 查询设备状态 (Feature=0x00, Cmd=0x0D, Payload=[07,00,00,00,04])
3. 查询配置项 (Feature=0x00, Cmd=0x0C, Payload=[配置ID])
4. 查询固件版本 (Feature=0x00, Cmd=0x05)
5. 查询 ANC 状态 (Feature=0x40, Cmd=0x03)
6. 查询 ANC 可用模式 (Feature=0x40, Cmd=0x29)
7. 查询 Gain 状态 (Feature=0x1E, Cmd=0x01)
8. 查询编解码器状态 (Feature=0x20, Cmd=0x05)
9. 查询设备管理信息 (Feature=0x1A, Cmd=0x00)
10. 查询配对设备信息 (Feature=0x28, Cmd=0x01/0x03/0x05)
11. 查询序列号 (Feature=0x00, Cmd=0x14)
12. 查询设备ID (Feature=0x00, Cmd=0x15)
13. 查询电量 (Feature=0x1A, Cmd=0x00)

### ANC 模式切换序列

```
TX: FF 04 00 01 00 1D 40 04 01  → 通透模式
RX: FF 04 00 01 00 1D 41 04 01  → 确认
TX: FF 04 00 01 00 1D 40 04 02  → 降噪模式
RX: FF 04 00 01 00 1D 41 04 02  → 确认
TX: FF 04 00 01 00 1D 40 04 00  → 关闭
RX: FF 04 00 01 00 1D 41 04 00  → 确认
```

### Gain 级别切换序列

```
TX: FF 04 00 01 00 1D 1E 02 01  → 中
RX: FF 04 00 01 00 1D 1F 02 01  → 确认
TX: FF 04 00 01 00 1D 1E 02 02  → 低
RX: FF 04 00 01 00 1D 1F 02 02  → 确认
TX: FF 04 00 01 00 1D 1E 02 00  → 高
RX: FF 04 00 01 00 1D 1F 02 00  → 确认
```

### 关键发现

1. **Feature/Cmd 字段**: 基础查询使用 Feature=0x00，Cmd 为功能号。ANC/Gain/Codec 使用独立 Feature ID。
2. **响应 Feature**: 响应 Feature = 请求 Feature | 0x01 (bit0)。
3. **Len 字段**: Len = 1 + 1 + payload.size（feature + cmd + payload）。
4. **心跳包**: 设备定期发送 Feature=0x01, Cmd=0x07 的空包作为心跳。
5. **设备管理**: Feature=0x1A 用于设备管理（电量、配对设备等），但通过 SPP 查询电量无响应。
